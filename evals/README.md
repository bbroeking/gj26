# gj26 evals

Validation pipeline for content (items, NPCs, enemies, story).

See `docs/content-evals-playbook.md` for the methodology.

## Layers

| File | Layer | Kind | Cost |
|---|---|---|---|
| `schema.eval.js` | 1 — schema validation | code | <1s |
| `tier-budget.eval.js` | 2 — tier conformance | code | <1s |
| `cross-refs.eval.js` | 3 — referential integrity | code | <1s |
| `voice.eval.js` | 4 — voice/style | LLM | ~$0.001 / item |
| `lore.eval.js` | 5 — lore consistency | LLM | ~$0.002 / item |
| `balance-sim.js` | 6 — balance | code (Monte Carlo) | ~5s |
| `soul-check.js` | 7 — design philosophy | LLM | ~$0.005 / feature |

## Run

```bash
# Cheap evals only — every commit
node evals/run-all.js

# Full evals including LLM graders — pre-release / monthly
node evals/run-all.js --full
```

## Cross-model rule

If content is generated by GPT, grade it with Claude (and vice versa). Never grade with the same family that generated.

## Auditing

LLM graders drift. Sample 10% of grader output to `audits/` monthly; review by hand. If grader-vs-human disagreement >20%, retune the rubric in `prompts/`.

## Traces

Generator runs log full prompt + output + eval scores to `traces/`. Useful for debugging when something ships broken.
